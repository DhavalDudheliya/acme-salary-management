What this feature does

When a support email arrives, the system automatically reads it, decides what kind of issue it is, attaches the right labels (tags), and then sends the ticket to the correct agent — all without anyone touching it manually. Admins define the rules; the AI does the reading.

Part 1 — The email arrives
1

Customer sends a support email

The customer writes an email to your support address — e.g. "Hi, I was charged twice this month and need a refund urgently." The system receives it automatically.

2

A ticket is created instantly

The system immediately creates a new ticket with the email subject, body, sender name, and timestamp. The ticket status is set to "New". This happens in under a second — no waiting for a human.

Part 2 — AI reads and tags the ticket
3

Gemini AI reads the email

In the background, the AI reads the full subject and body. It does not make up tags — it can only choose from a fixed list of tags that your team has pre-approved. It also gives a confidence score to each tag it picks.

4

Tags are chosen and attached

For the example email above, the AI might attach these tags:

Billing issue Refund request High priority Billing dept Frustrated SLA risk

Tags with a confidence score below 70% are marked as "suggested" — shown to the agent for confirmation rather than applied automatically.

Part 3 — The full tag library

All possible tags are grouped into categories. Agents and AI can only pick from this list — no free-form tags.

Part 4 — Assignment rules (admin sets these)

The admin creates rules in plain language. Each rule says: "If a ticket has these tags, assign it to this agent or team." Rules are checked in order — the first matching rule wins.

Example rules an admin might configure

1

If "Critical" AND "Bug" → assign to Senior Engineering team

Highest priority rule. Catches all production-down situations immediately.

2

If "Billing issue" OR "Refund request" → assign to Billing team

Any money-related ticket goes directly to the billing agents.

3

If "VIP customer" → assign to Account Manager + flag as urgent

VIP tickets bypass the queue and go straight to a named person.

4

If "General inquiry" → distribute round-robin to Support team

Low-priority tickets are evenly distributed so no agent gets overloaded.

5

No rule matched → goes to the unassigned queue

A team lead reviews and manually assigns it. This is the fallback for unusual cases.

Part 5 — What each person sees
Customer

Gets an automatic acknowledgement email: "We've received your request and someone will be in touch shortly." They never see the tags or the routing logic.

Agent

Opens their queue and sees the ticket already tagged and assigned to them. They can see which tags were applied and the AI's reasoning. They can add, remove, or override any tag manually if the AI got it wrong. If a tag was "suggested" (low confidence), they see a prompt asking them to confirm or reject it.

Admin

Has a dedicated "Rules" screen where they can create, reorder, enable/disable, and delete assignment rules — no code required. They can also see an "AI decisions log" that shows every ticket, what tags the AI picked, the confidence score, and which rule fired. This lets them tune both the tags and the rules over time.

Part 6 — Edge cases handled

AI is unsure

If confidence is below 70%, the tag is shown as a suggestion. The agent confirms or rejects it with one click.

No rule matches

Ticket lands in an "unassigned" queue visible to team leads. Nobody drops through the cracks.

Agent is offline

Rules can optionally check agent availability and fall back to assigning to the team instead of one person.

AI service is down

Ticket is created normally but goes straight to the unassigned queue without tags. No emails are lost.
